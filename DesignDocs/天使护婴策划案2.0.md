## 1. 设计理念

### 1.1 为什么类吸血鬼幸存者需要创新？

传统 bullet heaven 的循环是：**走位 → 输出 → 躲弹幕 → 再走位**。这个循环经过 Vampire Survivors、Brotato、Soulstone Survivors 三代验证，已经非常成熟——但也接近饱和。所有新作都在做同一件事：更多武器、更炫特效、更多数值。

我们要做的不只是"更多"。

### 1.2 核心创新点

**"永生天使 + 脆弱婴儿"的双层结构**重新定义了玩家关注的重心：

| | 传统 bullet heaven | 天使护婴 |
|---|---|---|
| 玩家关注 | 自身血量+位置 | 婴儿精神力+位置+情感状态 |
| 走位目标 | 躲避伤害 | 引导婴儿+趋近关键地形 |
| 失败原因 | 自己死了 | 婴儿精神力归零 |
| 构筑方向 | 输出+生存 | 三位一体（天使输出+婴儿保护+地形利用）|

### 1.3 我们的设计哲学

1. **保护比杀戮更有情感重量**——你不是为自己而战，你是一个守护者
2. **不确定性是设计语言，不是 bug**——婴儿不听指令不是缺陷，是游戏性
3. **地形是第三个角色**——它不只是一张地图，它是活的
4. **指数级爽是 earned 的**——前期你需要挣扎，后期你才配碾压

## 2. 核心体验支柱

### 2.1 体验目标

**你希望玩家在每个阶段感受到什么？**

| 阶段 | 时间 | 核心体验 | 关键机制 |
|------|------|---------|---------|
| 0-3分钟 | 理解 | "这婴儿怎么这么不听话？！" | 高好奇度婴儿、初始迷宫探索、前3波温和敌人 |
| 3-8分钟 | 掌控 | "我开始能引导TA了..." | 第一张路径卡、首次地形活化、忠诚度提升 |
| 8-15分钟 | 构筑 | "这个Build有意思" | 武器商店(10级)、融合卡牌出现、武器连携首次触发 |
| 15-25分钟 | 爆发 | "爽！满屏弹幕+婴儿觉醒！" | 指数成长曲线启动、多连携触发、地形觉醒 |
| 25分钟+ | 顶点 | "我是神" | 全机制解锁、婴儿觉醒+地形升华+全部连携 |

### 2.2 三层核心循环

```
秒级循环（操作反馈）
├── 天使攻击 → 击杀敌人 → 掉落经验/道具 → 即时正反馈
├── 婴儿AI移动 → 天使调整位置 → 地形利用 → 微决策
└── 武器连携触发 → 特效爆发 → 超预期快感

分钟级循环（短期目标）
├── 升级选卡 → 构筑强化 → 战局变化
├── 武器商店(每10级) → 新武器 → 新连携可能
├── 地形活化触发 → 战场环境变化 → 策略调整

整局循环（长线成长）
├── 婴儿从累赘→战神 → 情感弧线完成
├── 地形从静默→升华 → 战场叙事完成
└── 指数成长 → 后期碾压 → 终极满足感
```

## 3. 角色与属性体系

### 3.1 天使（玩家操控）

天使永生不死，是纯粹的输出机器。没有血量、没有防御需求——这种"去生存化"设计是游戏最大的差异化。

**属性体系**（详见 `角色属性2.0.xlsx → 天使属性-可见`；**隐藏属性**见 `天使属性-隐藏` Sheet）：

| 类别 | 核心属性 | 设计意图 |
|------|---------|---------|
| 战斗核心 | 武器伤害、攻击频率、暴击率/伤害、冷却缩减 | 输出构筑基础 |
| 投射物 | 数量、速度、大小、穿透、连锁 | 弹幕构筑维度 |
| 元素 | 冰冻/燃烧几率、范围伤害半径 | 元素交互基础 |
| 战术 | 吸引几率（转移仇恨）、移动速度 | 天使操作维度 |
| 成长 | 幸运、经验加成、武器连携加成 | 指数成长发动机 |

**关键设计决策**: 天使属性中的"吸引几率"是一个非常有趣的设计——你通过攻击概率性地将敌人仇恨从婴儿转移到自己身上。由于你不会受伤，这个属性的价值不是生存，而是**为婴儿买时间**。

### 3.2 婴儿（AI保护对象→成长后副C）

婴儿的属性体系代表了它从"脆弱核心"到"战斗伙伴"的成长弧线。

**属性体系**（详见 `角色属性2.0.xlsx → 婴儿属性-可见`）：

| 类别 | 核心属性 | 设计意图 |
|------|---------|---------|
| 生命 | 最大精神力、恢复速度、护盾 | 婴儿的生存底线 |
| 防御 | 伤害减免、生命传输 | 天使输出→婴儿生存的协同 |
| AI | 忠诚度、好奇度、移动速度 | 行为可控性vs不可预测性的平衡 |
| 情感 | 恐惧阈值、愤怒阈值、觉醒充能 | 情感状态机触发器 |
| 输出 | 婴儿攻击力、元素亲和 | 成长→副C的转化 |

**关键设计决策**: 忠诚度和好奇度是互补属性。忠诚度上限95%（永远留5%的不可预测空间），好奇度下限30%（不会变成完全跟随的跟屁虫）。这保证了**整个游戏过程婴儿始终有AI趣味**。

## 4. 婴儿AI与情感状态机

### 4.1 设计目标

用户反馈：**"不能太过于平缓简单，还是得不听话点"**。但也不能完全不可控导致挫败。

解决方案：**情感状态机**——婴儿不是随机乱跑，而是基于"情感"做决策。情感由战场状态自然驱动。

### 4.2 五种情感状态

```
                 ┌─────────────────────────────────────┐
                 │         情绪状态机 (每0.5秒刷新)        │
                 └─────────────────────────────────────┘

  附近敌人<恐惧阈值          ┌──────────┐        附近敌人≥恐惧阈值
  ────────────────────────▶ │   好奇    │ ◀─────────────── (安全后)
                             │ (默认)    │
                             │ 随机探索   │
                             └─────┬─────┘
                                   │
                   连续受伤≥愤怒阈值│          精神力<30%
                                   ▼                 ▼
                             ┌──────────┐    ┌──────────┐
                             │   愤怒    │    │   疲惫   │
                             │ 无敌1秒   │    │ 原地喘息  │
                             │ 暴走攻击  │    │ 回复×2   │
                             └─────┬─────┘    └────┬─────┘
                                   │               │
                          觉醒充能≥100│               │精神力>30%
                                   ▼               ▼
                             ┌──────────┐    ┌──────────┐
                             │   觉醒    │    │   好奇    │
                             │ 终极变身  │    │ (恢复)    │
                             └──────────┘    └──────────┘
```

**参数详见** `角色属性2.0.xlsx → 情感状态机`

### 4.3 行为树实现（伪代码）

```
ROOT (并行选择器)
├── [条件: 觉醒充能 ≥ 100] → 觉醒状态 (最高优先级)
├── [条件: 连续受伤 ≥ 愤怒阈值] → 愤怒状态
├── [条件: 精神力 < 30%] → 疲惫状态
├── [条件: 附近敌人 ≥ 恐惧阈值] → 恐惧状态
└── [默认] → 好奇状态 (最低优先级)

好奇状态:
├── 序列: 判定方向
│   ├── 加权随机: 忠诚度%→跟随天使, 好奇度%→探索随机方向
│   ├── [条件: 距离天使 > 最大距离] → 强制向天使移动
│   ├── 路径检测: 验证目标位置可达(不穿墙)
│   └── 执行移动
└── 序列: 探索行为
    ├── 10%概率: 原地停留1-2秒
    ├── 新区域发现: 显形区域内敌人2秒
    └── 靠近可破坏物: 50%概率尝试破坏
```

### 4.4 前期不"平缓"但也不"崩溃"的设计

| 波次 | 敌人强度 | 婴儿初始情感 | 设计意图 |
|------|---------|-------------|---------|
| 1-2波 | 2-3只缓慢僵尸 | 好奇度高 | 玩家看到婴儿不听话的"麻烦"，建立对保护对象的情感 |
| 3-5波 | 4-6只，出现远程 | 首次触发恐惧 | 婴儿第一次求救→玩家第一次感到"我必须保护TA" |
| 6波+ | 混合兵种 | 情感状态活跃切换 | 情感节奏建立，玩家适应并开始利用情感系统 |

## 7. 武器连携系统

> 📂 **开发参阅**：`武器系统.xlsx → Buff标签兼容矩阵` + `角色属性2.0.xlsx → 武器连携矩阵`（8组连携详细参数、兼容性分数矩阵）

### 7.1 设计理念

武器不是独立的DPS单元，它们之间会产生化学反应。"两把武器比两把武器总和多一些"——这"多一些"就是构筑深度。

### 7.2 连携矩阵（8组）

| 连携 | 武器组合 | 核心机制 | 冷却 |
|------|---------|---------|------|
| 蒸汽爆炸 | 烈焰风暴+冰霜新星 | 双元素命中→150%混合伤害+眩晕 | 8s |
| 光棘螺旋 | 圣光之环+荆棘之环 | 双环绕接近→范围+40%，流血×2 | 15s |
| 弹幕同步 | 圣光连弩+圣光霰弹 | 双弹命中→3秒全远程攻速+50% | 12s |
| 雷火交加 | 雷电之链+烈焰风暴 | 弹跳链被爆炸覆盖→弹跳翻倍+灼烧 | 6s |
| 神圣壁垒 | 守护之盾+圣光之环 | 盾击命中→环绕护婴儿+减伤 | 20s |
| 冰封绝境 | 冰霜新星+圣光审判 | 冰冻后审判→冰冻敌人伤害×3 | 30s |
| 风暴之眼 | 天使刺剑+雷电之链 | 三连刺全中→雷电风暴 | 10s |
| 圣痕追击 | 追踪圣光+天使步枪 | 追踪→穿透追伤+100% | 5s |

**详见** `角色属性2.0.xlsx → 武器连携矩阵`

### 7.3 连携触发机制

每把武器攻击时，向全局连携累加器贡献计数（权重 = 武器稀有度）：

- 普通武器: +1
- 稀有武器: +2
- 史诗武器: +3
- 传说武器: +5

累计达到100时，从已拥有的武器对中随机触发一组已解锁连携。这个设计保证：
- 持有越多武器，连携越容易触发
- 持有越稀有武器，连携触发越快（指数成长的发动机之一）

## 8. 地形生成算法

> 📂 **开发参阅**：`开发参考-地形生成算法.md`（BSP伪代码、Delaunay三角剖分+MST、质量验证修复流程）+ `角色属性2.0.xlsx → 地形生成算法`（16项可调参数）

### 8.1 总纲

采用**BSP（二叉空间分区）+ 约束满足 + 图论验证**的三步生成法。

### 8.2 步骤一：BSP 房间生成

```
地图: 3000×3000px

1. 从根节点开始BSP递归分割（最小房间 400×400px）
2. 每步分割随机选择横切/竖切，偏移±15%(避免均匀)
3. 生成 8-14 个叶节点房间
4. 删除面积<最小阈值的房间（但保证 ≥ 8个房间）
```

### 8.3 步骤二：走廊连接

```
1. 构建房间间的 Delaunay 三角剖分
2. 从三角剖分中提取最小生成树（MST）——保证所有房间连通
3. 随机回填 30-50% 的非MST边作为额外走廊（形成环路）
4. 走廊宽度: 120-180px（随机）
5. 走廊路径: L形（避免直通狙击线）
```

### 8.4 步骤三：质量验证与修复

```
验证清单:
□ 连通性: 从任意房间能否到达任意房间（BFS验证）
□ 环形走廊: 是否存在≥1条闭环（拓扑检测）
□ 安全房间: 是否存在≥1个只有1个出入口的口袋房间
□ 初始区: 出生点周围300px内无可生成敌人的区域
□ 门分布: ≥30%的走廊连接处有门（可开关的物理分隔）

修复:
- 无环形走廊 → 回填最近的非MST边
- 无口袋房间 → 随机选取1个房间，删除其最远走廊连接
- 门太少 → 在未设门的走廊随机补门
```

### 8.5 可配置参数

| 参数 | 默认 | 范围 | 说明 |
|------|------|------|------|
| mapSize | 3000 | [2000,5000] | 地图边长 |
| min/maxRooms | 8-14 | [5-25] | 房间数量 |
| corridorWidth | 150 | [80-300] | 走廊宽度 |
| safeRoomMin | 1 | [1,3] | 口袋房保底 |
| loopMin | 1 | [0,3] | 环形走廊保底 |
| destructibleDensity | 0.03 | [0.01-0.08] | 可破坏物密度 |

**详见** `角色属性2.0.xlsx → 地形生成算法`

## 9. 地形活化系统

> 📂 **开发参阅**：`角色属性2.0.xlsx → 地形活化系统`（5阶段演化参数、随机灾变事件表）

### 9.1 设计理念

地形不是死的地图——它随着战局演化。借鉴《Hades》的房间变化和《Noita》的环境互动，但适配俯视角+类吸血鬼的节奏。

### 9.2 五阶段演化

```
阶段0: 静默 ──→ 阶段1: 警觉 ──→ 阶段2: 涌动 ──→ 阶段3: 觉醒 ──→ 阶段4: 升华
(开局)      (精神力<60%)    (杀敌≥30)     (杀敌≥80+精神力>50%)  (婴儿觉醒触发)
```

| 阶段 | 触发 | 地形效果 | 玩法影响 |
|------|------|---------|---------|
| 静默 | 开局 | 初始迷宫 | 学习地形 |
| 警觉 | 精神力<60% | 墙壁生长减速藤蔓(15%) | 自动保护启动 |
| 涌动 | 杀敌30+ | 地面圣光纹路(移速+30%) | 奖励击杀效率 |
| 觉醒 | 杀敌80+精神力>50% | 地图边缘崩塌缩小 | 加速节奏 |
| 升华 | 婴儿觉醒 | 全屏圣光+全敌减速 | 终极形态 |

### 9.3 随机灾变事件

每45-90秒随机触发一次选择事件，打破固定节奏：

- **天降光柱**: 地图中心出现秒杀光柱（可推敌人进入）
- **毒沼扩散**: 边缘出现毒沼，压缩战斗空间
- **祝福泉涌**: 随机出现3个治疗泉
- **时间扭曲**: 全场时间流速波动(±30%)

## 11. 指数成长设计

> 📂 **开发参阅**：`角色属性2.0.xlsx → 指数成长公式`（7阶段属性/敌人/掉落成长曲线数学公式）

### 11.1 用户的明确需求

> **"我希望这个游戏到后期是可以指数级爽的"**

这意味着我们需要设计一个**前期受控、中期加速、后期爆炸**的递增曲线。

### 11.2 指数成长的多重发动机

| 发动机 | 机制 | 指数性从何而来 |
|--------|------|---------------|
| **F04圣光灌注** | 婴儿击杀→天使攻击力+1(永久) | 攻击力↑→婴儿伤害↑→更多击杀→更多攻击力 |
| **武器连携** | 稀有武器→更快触发→更频繁爆发 | 更多连携→更快击杀→更快获取武器→更多连携 |
| **属性宝箱** | 指数成长系数×宝箱增益量 | 属性总量越高→宝箱增益倍率越高 |
| **地形活化** | 杀敌→地形进化→更多伤害加成 | 地形帮杀敌→杀敌推进地形→地形帮杀更多 |
| **婴儿觉醒** | 觉醒越频密→持续时间越长 | 后期觉醒覆盖率可达40%+ |

### 11.3 防止失控的软限制

- 弹幕数量硬上限45颗（性能+视觉可读性）
- 武器数量上限6把（决策空间）
- 连携冷却最低1秒（操作感保留）
- 敌人强度同步指数增长（对抗保持紧张）

**详见** `角色属性2.0.xlsx → 指数成长公式`
