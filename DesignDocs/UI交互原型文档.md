# 天使护婴 - UI/UX交互原型文档

> **文档版本**：v2.0  
> **用途**：供UI/UX设计师参考，实现全部界面交互  
> **配套文件**：`天使护婴策划案2.0.md`、`技能系统.xlsx`、`武器系统.xlsx`、`角色属性2.0.xlsx`

---

## 一、整体设计语言

### 1.1 视觉风格

| 项目 | 规范 |
|------|------|
| 画风 | 手绘厚涂 × 天使光翼元素 |
| 主色调 | 金色（天使）+ 粉色（婴儿）+ 暗紫（地狱） |
| 字体 | 中文：思源黑体 / 英文：Cinzel（古典衬线） |
| UI边框 | 金色纹样边框，天使羽毛装饰 |
| 动画曲线 | 缓出（ease-out）为主，卡牌选择用弹性动画 |

### 1.2 布局原则

- **信息密度**：手机屏幕同时显示元素 ≤ 8个
- **手指热区**：按钮最小 44×44pt，间距 ≥ 8pt
- **可读性**：战斗中血量/精神力数字字号 ≥ 18pt
- **层次感**：Z轴顺序：地形 > 敌人/子弹 > 角色 > 特效 > UI

### 2.3 核心HUD元素详细规范

#### 婴儿精神力条 ⚠️ 最重要的UI元素

| 项目 | 规范 |
|------|------|
| 位置 | 屏幕顶部，居中 |
| 尺寸 | 宽度60%屏幕，高度12pt |
| 颜色 | 绿(100-70%) → 黄(70-40%) → 红(40-0%) |
| 动画 | 下降时红色脉冲，归零前5秒红色闪烁 + 警报音效 |
| 数字显示 | 右侧显示百分比（如 67%），字号18pt，白色描边 |

**设计意图**：精神力条是玩家最紧张的来源，必须**一眼可见**，且归零前有强烈预警。

#### 波次指示器

| 项目 | 规范 |
|------|------|
| 位置 | 精神力条下方，居中 |
| 格式 | "WAVE 7 / 25" |
| 字体 | 14pt，金色，有描边 |
| 新波次提示 | 全屏淡入淡出 "WAVE 8" 1.5秒 |

#### 武器栏

| 项目 | 规范 |
|------|------|
| 位置 | 屏幕底部 |
| 数量 | 最多4把武器同时携带 |
| 显示 | 武器Icon(48×48pt) + 弹药数（远程武器）|
| 切换 | 点击武器Icon切换，当前武器有金色边框 |
| 武器特效 | 连携触发时，相关武器Icon有颜色光效闪烁 |

#### 卡牌状态栏

| 项目 | 规范 |
|------|------|
| 位置 | 武器栏上方 |
| 内容 | 当前携带卡牌数 "7/10"，已满时红色警告 |
| 光环指示器 | 当前激活的光环Icon（小，24×24pt）|
| 连携指示器 | 当前激活的连携名称 + 剩余时间进度条 |

## 三、升级选卡界面 ⭐ 核心交互

- 天使升级时（击杀足够敌人）

### 3.4 重roll机制交互

| 交互步骤 | 表现 |
|---------|------|
| 1. 点击"重roll"按钮 | 按钮旋转动画，消耗经验值动画（数字减少） |
| 2. 3张卡牌重新随机 | 翻牌动画（0.4秒） |
| 3. 新卡牌出现 | 新卡牌有金色光效0.5秒 |
| 限制 | 每级最多重roll 2次，Meta永久解锁后无限制 |

## 九、动画规范

### 9.1 动画时长基准

| 动画类型 | 时长 | 缓动函数 |
|---------|------|---------|
| 卡牌翻面 | 0.4秒 | ease-out |
| 卡牌选择确认 | 0.3秒 | ease-out |
| 武器切换 | 0.2秒 | linear |
| 界面弹出 | 0.3秒 | ease-out + 轻微缩放 |
| 界面关闭 | 0.2秒 | ease-in |
| 连携触发 | 0.5秒 | 无缓动（关键帧动画）|
| 升级闪光 | 0.8秒 | ease-out |
| 失败碎裂 | 1.5秒 | 物理模拟 |

### 9.2 动画性能预算

## 十三、开发实现注意事项

### 13.1 性能优化

1. UI元素超过20个时，使用对象池复用
2. 卡牌Icon在滚动列表中需懒加载
3. 失败界面的Meta进度动画需分帧加载，避免卡顿

### 13.2 可访问性

1. 颜色盲模式：增加图案区分（而不仅靠颜色）
2. 字体大小可调：设置中可调整UI字号（0.8x-1.5x）
3. 震动反馈：关键操作（升级、连携、失败）有震动反馈（手机）

### 13.3 本地化

1. 所有文本从JSON读取，不直接写死在UI里
2. 中文、英文、日文文本长度差异：UI元素需支持动态宽度
3. 图标优先：跨文化理解的Icon优先于文字
